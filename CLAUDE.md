## 项目定位

本项目是一个**可运行、可演示、可迭代的网页无障碍增强原型系统**，目标不是构建工业级浏览器插件，而是在比赛周期内完成一个能够稳定展示核心价值的 Demo。

核心目标：

1. 提升视障用户对网页的**内容理解能力**
    
2. 降低老年用户与临时障碍用户的**认知负担**
    
3. 在**不修改原网页源代码**的前提下，通过注入式增强实现功能

---

## 开发策略（时间紧迫）

**核心原则：假数据 + 硬编码 + 演示优先**

由于时间限制，放弃通用方案，采用以下策略：

1. **平台特定适配**
   - 只适配今日头条（首页 + 文章详情页）
   - 选择器、样式、逻辑全部硬编码
   - 不考虑其他网站的兼容性

2. **假数据优先**
   - 图片语义增强：预设几张图片的描述，匹配 URL 或位置直接返回
   - 页面摘要：针对特定文章 URL 返回预设摘要
   - AI 调用作为兜底，但演示时选择预设好的页面

3. **演示路径固定**
   - 准备 2-3 个固定的头条文章 URL
   - 这些文章的所有功能都经过测试和优化
   - 演示视频只在这些页面上录制

4. **不追求完美**
   - 边界情况不处理（如网络错误、API 超时）
   - 只保证演示路径 100% 可用
   - 其他场景能跑就行，不能跑也接受

## 技术选型原则

所有技术选型遵循以下原则：

- **优先可实现，不优先炫技**
    
- **优先演示稳定，不优先功能全面**
    
- **优先前端注入式方案，不引入复杂后端**
    
- **优先最小闭环，再逐步扩展模块**
    
- **AI 只用于"语义增强"，不承担全部页面理解任务**
    

---

## 总体技术架构

```text
网页
↓
前端注入层（JavaScript）
↓
AI语义处理层（DeepSeek API）
↓
DOM增强 / UI面板展示
↓
用户 / 读屏软件
```

---

## 技术栈

### 前端

- **HTML**
    
- **CSS**
    
- **Vanilla JavaScript**
    

选择理由：

- 学习和实现成本最低
    
- 适合快速构建可演示原型
    
- 不依赖复杂框架，便于 Claude Code 快速生成与修改
    

---

### AI 接口

- **DeepSeek API**
    

使用方式：

- 页面摘要：发送提取后的网页文本给 DeepSeek
    
- 图片语义：先支持"演示级描述"，后续视情况接入图像理解能力
    

注意事项：

- DeepSeek 主要承担**文本理解与生成**
    
- 若图像理解接入成本过高，可在比赛版本中采用：
    
    - 模拟语义描述
        
    - 半自动描述
        
    - 固定测试样例描述
        

---

### 页面增强方式

- **DOM 读取**
    
- **DOM 注入**
    
- **CSS 样式覆盖**
    
- **元素隐藏 / 重排**
    

选择理由：

- 不需要改造源站
    
- 符合"无障碍增强层"概念
    
- 易于实现"前后对比"演示
    

---

## 不采用的技术

以下技术在比赛原型阶段**明确不采用**：

- 不使用后端数据库
    
- 不做用户系统
    
- 不做复杂浏览器插件打包流程
    
- 不做通用广告识别模型
    
- 不做工业级图像识别流水线
    
- 不引入 React / Vue / TypeScript，除非后续确有必要
    
- 不追求多页面工程结构的复杂设计
    

原因：

- 会显著增加开发成本
    
- 与当前比赛阶段目标不匹配
    
- 容易导致"工程复杂度大于产品价值"

## 模块实现策略

### 1. 页面摘要

技术路线：

- 提取 `document.body.innerText`
    
- 对文本进行截断和清洗
    
- 调用 DeepSeek API
    
- 返回简洁摘要并展示到控制面板
    

目标：

- 帮助用户快速理解"当前页面是什么、主要内容是什么、能做什么"
    

---

### 2. 图片语义

技术路线：

- 获取页面中的主要图片元素
    
- 优先处理有代表性的图片
    
- 将描述写入 `alt` 或旁路描述面板
    

注意：

- 比赛版本不追求全站图片100%自动理解
    
- 重点演示"从无语义到有语义"的改善
    

---

### 3. 信息提取与简化展示

技术路线：

- 读取标题、正文、关键段落
    
- 生成简化阅读区
    
- 隐藏非核心区域
    

注意：

- 不做"精准广告识别"
    
- 只做"信息复杂度降低"
    

---

### 4. 专注模式 / 降噪

技术路线：

- 隐藏导航、侧栏、页脚、浮层等常见干扰区域
    
- 提供恢复按钮
    
- 通过样式优化增强阅读体验
